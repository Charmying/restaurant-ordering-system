# 維運手冊

[English](./OPERATIONS.md) | **繁體中文**

最後驗證：2026-02-17

本手冊定義後端部署與事件回應的標準維運程序。

---

## 服務資訊

- 執行時：Node.js (NestJS)
- API 前綴：`/api`
- Swagger：`/api/docs`
- 即時命名空間：`/events`
- 主要資料儲存：MongoDB

---

## 部署前檢查清單

1. `npm ci` 或 `npm install` 成功
2. `npm run build` 成功
3. 必要的環境變數存在且有效
4. 資料庫連線測試通過
5. 檢視環境的種子行為 (特別是 superadmin 憑證)
6. 前一版本的回滾產物可用

---

## 部署程序

1. 建置產物：
    - `npm run build`
2. 啟動程序：
    - `npm run start:prod`
3. 驗證健康：
    - `GET /api/health`
4. 驗證認證路徑：
    - login + verify 端點
5. 驗證一個寫入路徑：
    - 在非關鍵領域的建立/更新操作

---

## 部署後冒煙測試

- API 健康端點回傳成功
- Swagger 載入且關鍵端點可見
- 登入與 token 刷新運作
- 至少一個 manager 保護的端點回傳預期的認證行為
- WebSocket 客戶端可連線至 `/events`

---

## 事件回應

### 嚴重性模型
- Sev-1：完全中斷或資料完整性風險
- Sev-2：主要功能不可用但有替代方案
- Sev-3：輕微降級

### 前 10 分鐘
1. 確認當前症狀與影響範圍
2. 凍結非必要部署
3. 檢查最近的部署與配置變更
4. 驗證資料庫連線與認證 secrets 有效性
5. 決定回滾或熱修復路徑

### 常見失敗模式
- **啟動失敗**：環境變數驗證錯誤或資料庫 URI 問題
- **認證激增**：無效的 JWT secrets、token 不匹配或時鐘偏移
- **即時問題**：命名空間不匹配 (`/events`) 或客戶端重連迴圈
- **資料異常**：訂單/桌位生命週期中的不正確狀態轉換

---

## 回滾策略

1. 回復到最後已知良好的產物
2. 如果配置導致問題，恢復先前的環境變數配置集
3. 重新執行冒煙測試
4. 記錄根本原因並新增預防措施項目

---

## 資料安全注意事項

1. `POST /api/orders/reset` 具破壞性且僅限 superadmin
2. `DELETE /api/messages/all` 具破壞性且僅限 superadmin
3. 對於緊急腳本，在生產環境需要雙重批准

---

## 可觀察性基準 (建議)

最小指標儀表化：

- 請求率與延遲 (依路由 + 方法)
- 依狀態碼類別的錯誤率
- 認證失敗 (401/403)
- websocket 連線數
- MongoDB 操作延遲

最小日誌：

- 啟動配置驗證結果
- 未處理的例外 (含堆疊)
- 認證與權限拒絕原因類別 (不含 secrets)
