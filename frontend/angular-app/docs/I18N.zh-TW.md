# i18n 設定指南

[English](./I18N.md) | **繁體中文**

前端使用 `@ngx-translate/core` 搭配 `public/assets/i18n` 下的 JSON 翻譯資源。

---

## 快速開始

### 在元件中
```typescript
import { TranslateModule } from '@ngx-translate/core';

@Component({
  imports: [TranslateModule],
  template: `<h1>{{ 'layout.header.title' | translate }}</h1>`
})
export class MyComponent {}
```

### 程式化翻譯
```typescript
import { TranslateService } from '@ngx-translate/core';

export class MyComponent {
  private translate = inject(TranslateService);

  get title(): string {
    return this.translate.instant('layout.header.title');
  }
}
```

---

## 執行時整合

`TranslateModule` 在 `src/app/app.config.ts` 中配置 JSON 載入器：

- 翻譯檔案從 `/assets/i18n/{lang}.json` 載入
- `fallbackLang` 設定為 `zh`
- 語言選擇由 `LanguageService` 控制 (目前在 `zh` 與 `en` 之間切換)

`LanguageService` 將活動語言持久化於 `localStorage` 的 `app_lang` 鍵下。

---

## 檔案結構

```text
frontend/angular-app/
└── public/
    └── assets/
        └── i18n/
            ├── en.json
            └── zh.json
```

---

## 新增鍵

1. 將鍵同時新增至 **en.json** 和 **zh.json**
2. 遵循 `../../../docs/architecture/i18n-naming.zh-TW.md` 中的命名慣例
3. 使用結構化鍵 (基於功能)，而非基於文字的鍵

---

## 配置參考

載入器配置請參閱 `src/app/app.config.ts`，語言持久化請參閱 `src/app/core/services/language.service.ts`。

---

## 維護

- 永遠同時將鍵新增至**所有**語言檔案
- 保持所有語言的鍵結構相同
- 遵循 `../../../docs/architecture/i18n-naming.zh-TW.md` 中的命名規則
- 提交前驗證 JSON 語法
